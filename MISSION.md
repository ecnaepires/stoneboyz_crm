# Mission: Learn To Code By Building The Price List Builder

## Why

The user wants to learn how to code by pairing on real product work in this CRM. The immediate goal is to understand and implement the Price List Builder instead of treating code changes as a black box.

## Success looks like

- The user can explain how a request moves from browser UI to API to domain logic to database.
- The user can make small TypeScript/React changes with guidance.
- The user can read tests, run them, and understand why they pass or fail.
- The user can describe the Price List Builder data model in plain language.

## Constraints

- Beginner-friendly pacing.
- Explain every important step before editing code.
- Use small changes with fast verification.
- Avoid unrelated refactors and jargon.

## Out of scope

- Full inventory system.
- Drawing workspace changes.
- Advanced framework theory not needed for this feature.
